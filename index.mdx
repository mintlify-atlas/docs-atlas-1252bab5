Development Containers (dev containers) are a standard for defining reproducible development environments inside containers. By placing a `devcontainer.json` file in your repository, you and your team can spin up identical, fully-configured environments — locally or in the cloud — without "works on my machine" problems.

## Why dev containers?

Dev containers solve the challenge of inconsistent development environments. Instead of writing setup READMEs that become outdated, you encode your environment in a `devcontainer.json` file that any supporting tool can use to automatically create an identical container.

## Quick start

<Steps>
  <Step title="Install the Dev Container CLI">
    Install the reference implementation CLI via npm:
    ```bash
    npm install -g @devcontainers/cli
    ```
  </Step>
  <Step title="Add a devcontainer.json to your project">
    Create a `.devcontainer/devcontainer.json` file referencing a base image:
    ```json .devcontainer/devcontainer.json
    {
      "name": "My Project",
      "image": "mcr.microsoft.com/devcontainers/base:ubuntu",
      "features": {
        "ghcr.io/devcontainers/features/node:1": {}
      },
      "postCreateCommand": "npm install"
    }
    ```
  </Step>
  <Step title="Start the dev container">
    Run the `up` command from your project folder:
    ```bash
    devcontainer up --workspace-folder .
    ```
  </Step>
  <Step title="Execute commands inside the container">
    Use `exec` to run commands in the running dev container:
    ```bash
    devcontainer exec --workspace-folder . npm test
    ```
  </Step>
</Steps>
